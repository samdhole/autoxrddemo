# XRD Batch Analysis Pipeline

## Goal
Portfolio piece for samdhole.github.io — "Scientific Automation" service lane.
Demonstrates: "A characterization lab runs 8 batches of the same material. A scientist
opens each pattern manually, identifies every reflection, records FWHM and position,
computes d-spacings by hand, writes a summary. This pipeline auto-detects all
reflections, tracks how peak positions and widths evolve across the batch, and delivers
a formatted technical memo — the same output, in under 20 seconds."

## Demo Dataset
Primary: `data/synthetic_quartz/` — 8 simulated quartz annealing patterns with
systematic FWHM broadening (0.08° → 0.22°). Generated by `generate_synthetic.py` from
ICDD 00-046-1045 quartz peak list, SNR=100, 3500 points over 10–80°. CLI defaults here.

Secondary: `data/xrd_samples/` — 8 diverse RRUFF minerals (Quartz, Calcite, Corundum,
Fluorite, Magnetite, Perovskite, Rutile, Kaolinite) for "first-look on unknown sample"
testing. Diverse phases — not a representative same-material batch.

## RRUFF File Format
- Headers: lines beginning with `##` (variable count, NOT fixed 11 lines)
- Pattern: `##FIELDNAME=VALUE`
- Key fields: NAMES, RRUFFID, X-RAY WAVELENGTH (may be absent → default 1.54056 Å)
- Data: whitespace or comma-separated `2θ  intensity` pairs
- Bulk download: https://www.rruff.net/zipped_data_files/powder/XY_RAW.zip

## Key Implementation Rules
1. Loader skips all `#`-prefixed lines; never hardcodes line count
2. Wavelength read from header; default 1.54056 Å if absent
3. Scherrer: D = (0.9 × λ) / (β_rad × cos(θ_rad)), θ = peak_center_2θ/2
4. β must be in radians (np.deg2rad(fwhm_deg))
5. Results are upper-bound estimates (instrument broadening not subtracted)
6. Detection: DoG (Difference of Gaussians) — narrow Gaussian (σ_signal_deg=0.04) for denoised signal, wide Gaussian (σ_baseline_deg=1.5°) for baseline. Peak = where bandpass exceeds k_sigma × bandpass-MAD-noise.
7. Fit: per-peak ROI (±0.25°) with single PseudoVoigt + linear background. Composite reconstruction by summing PVs + refined baseline (smoothed peak-subtracted signal).
8. Quality filter: drop fits with FWHM<0.04, R²<0.5, amplitude<0.25.
9. lmfit: PseudoVoigtModel, prefix='p_' per ROI, fraction param = η
10. Outlier: R² threshold + IQR; single Flag column with R² value embedded (e.g. "Poor fit (R²=0.830)")
11. All figures base64-embedded in single HTML file (no external deps)

## Key Constants
- `analyzer.CRYSTALLITE_SIZE_CAP_NM = 200.0` — Scherrer upper bound, flags "Instrument-limited"
- `analyzer.POOR_FIT_R2_THRESHOLD = 0.70` — flags "Poor fit (R²=X.XXX)". Calibrated for the unweighted-mean-of-local-R² metric; minor reflections at noise floor naturally pull the mean into 0.7-0.85 range.
- `fitter.SIGMA_SIGNAL_DEG = 0.04` / `SIGMA_BASELINE_DEG = 1.5` / `K_SIGMA = 4.0` — DoG detection (k_sigma multiplies MAD of bandpass lower-half, not a strict Gaussian σ)
- `fitter.ROI_WINDOW_DEG = 0.25` — per-peak ROI half-width
- `fitter.MIN_FIT_FWHM = 0.04` / `MIN_FIT_R2 = 0.5` / `MIN_FIT_AMP_FRAC = 0.0025` — quality filter (amplitude threshold is fraction of y.max(), so it scales with normalization)
- `loader.MAX_POINTS = 4000` — at 0.02°/pt typical step
- `r_squared` semantics: unweighted mean of per-peak local R², not a composite-vs-data R² (which would be self-referential since baseline is computed from peak-subtracted residuals). AIC/BIC are display-only and use k=6×n_peaks (4 PV + 2 linear bg per ROI).

## Known Issues
- Kaolinite R² < 0.95 expected (turbostratic disorder) — not a bug
- Z-score flagging removed — at n=8 it never fires; IQR + R² threshold is sufficient
- lmfit `fraction` param = η (Lorentzian mixing fraction) in PseudoVoigtModel
- lmfit PseudoVoigt: `fwhm = 2.0 * sigma` (NOT the Gaussian 2.3548 × sigma)
- RRUFF header line count varies (8–18), never fixed at 11
- Windows cp1252 terminals: `__main__.py` calls `sys.stdout.reconfigure(encoding="utf-8")` on startup
